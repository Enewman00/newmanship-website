### 2026-05-12 — Task 1.6: Deployment pipeline

**Status:** Complete

**What was done:**
- Created `public/_headers` for Cloudflare Pages with cache-control rules:
  - `/_astro/*` and `/fonts/*` → `Cache-Control: public, max-age=31536000, immutable` (content-hashed assets, safe to cache forever)
  - `/*` HTML → `max-age=0, must-revalidate` (ensure deploys propagate immediately)
  - `/rss.xml` → 1-hour TTL; `/sitemap-index.xml`, `/robots.txt` → 1-day TTL
  - Security headers added to all routes: `X-Content-Type-Options`, `X-Frame-Options: DENY`, `Referrer-Policy`
- Created `.github/workflows/lighthouse.yml` — runs Lighthouse CI on every PR and push to main
- Created `.lighthouserc.json` — targets all 5 key pages, enforces 95+ score on performance, accessibility, best-practices, and SEO; uses `temporary-public-storage` upload for PR report links

**Note:** Connecting the GitHub repo to a Cloudflare Pages project is a one-time manual step in the Cloudflare dashboard (set build command: `npm run build`, output dir: `dist`).
